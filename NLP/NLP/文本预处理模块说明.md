# NLP 文本预处理模块说明

## 模块位置

- **核心代码**：`preprocessing.py`
- **停用词表**：`stopwords.txt`

## 依赖

| 包名 | 用途 |
|------|------|
| `jieba` | 中文分词 |
| `opencc-python-reimplemented` | 繁简转换 |

## 快速使用

```python
from preprocessing import TextPreprocessor

pp = TextPreprocessor()
result = pp.process("#突发#！！某地暴雨太大了！！！详情见 http://xxx.com")
# result → "突发 某地 暴雨 太大 详情 见"
```

### 批量处理

```python
from preprocessing import batch_process, deduplicate_corpus

texts = ["文本1", "文本2", ...]
results = batch_process(texts)

# 带去重
deduped = deduplicate_corpus(texts, threshold=0.85)
```

### 分步调试

```python
steps = pp.process_with_steps("你的文本")
for step_name, output in steps.items():
    print(f"{step_name}: {output}")
```

## 处理流水线

共 **14 步**，按顺序执行：

```
原始文本
  ↓ ① 去除 HTML 实体    — html.unescape() 还原 &amp; &lt; &gt; 等
  ↓ ② 去除 HTML 标签    — 正则 <[^>]+>
  ↓ ③ 去除 URL          — 匹配 http/https 链接
  ↓ ④ 去除 @用户         — 匹配 @[\w\-_]+
  ↓ ⑤ #话题# 处理       — 去掉 # 号，保留话题文字
  ↓ ⑥ 去除表情符号       — Unicode 区块精准匹配（不含中日韩文字区）
  ↓ ⑦ 去除微博垃圾后缀  — "收起d""查看图片""转发微博" 等
  ↓ ⑧ 去除无意义符号    — 重复标点压缩 + 噪字符清理
  ↓ ⑨ 压缩空白          — 多空白 → 单空格
  ↓ ⑩ 广告检测          — 命中 ≥2 个关键词则丢弃整条
  ↓ ⑪ 繁→简转换         — OpenCC(t2s)
  ↓ ⑫ 分词              — jieba.cut (精确模式)
  ↓ ⑬ 停用词过滤        — 过滤 631 个停用词 + 单字非中文
  ↓ ⑭ 连续重复去重      — 相邻相同 token 合并
  输出: 空格分隔的关键词序列
```

## 各步骤详解

### 1-2. HTML 处理

```python
_unescape_html("&lt;br/&gt;")  # → "<br/>"
_remove_html_tags("<p>正文</p>")  # → "正文"
```

### 3-5. 微博特有格式

| 原始 | 处理后 |
|------|--------|
| `http://xxx.com` | （删除） |
| `@人民日报` | （删除） |
| `#突发#` | `突发` |

### 6. 表情符号

移除以下 Unicode 区段中的符号：

| 区块 | 范围 |
|------|------|
| Emoticons | U+1F600–U+1F64F |
| Misc Symbols | U+1F300–U+1F5FF |
| Transport | U+1F680–U+1F6FF |
| Flags | U+1F1E0–U+1F1FF |
| Supplement | U+1F900–U+1FAFF |
| Dingbats | U+2702–U+27B0 |
| 杂项符号 | U+2600–U+27BF |
| 其他散落符号 | ★⭐♻✅ 等 |
| 零宽字符 | ZWJ、零宽空格等 |

> **注意**：已排除范围 `\U000024C2-\U0001F251`（该范围覆盖了整个 CJK 统一表意文字区，会误删中文）。

### 7. 微博垃圾后缀

匹配并移除微博常见后缀：

- `收起d`、`收起`
- `查看图片`、`查看视频`、`查看全文`、`查看详情`
- `O网页链接`
- `转发微博`
- `我分享了`
- `（分享自...）`、`(分享自...)`

### 8. 无意义符号

两步处理：

1. **重复标点压缩**：`！！！` → `！`，`？？` → `？`
2. **噪字符清理**：移除非中文、非字母数字、非常用标点的孤立符号

### 9. 广告检测

精简后的关键词列表（**需 ≥2 次命中**才算广告）：

```
点击购买, 立即下单, 限时优惠, 免费领取, 加V,
添加微信, 扫码, 二维码, 转发抽奖, 关注送,
复制链接, 包邮, 清仓, 跳转
```

已从列表中移除易误判词：`广告`、`推广`、`赞助`、`淘宝`、`天猫`、`京东`、`拼多多`、`小程序`。

### 10. 繁简转换

使用 `OpenCC("t2s")` 将繁体中文转为简体。

### 11. 分词

采用 **jieba.cut（精确模式）**，不开启全模式或搜索引擎模式。

### 12. 停用词过滤

- 过滤 `stopwords.txt` 中定义的 631 个停用词
- 过滤纯标点/数字 token
- 过滤长度 < 2 的非中文字符（保留如"大""红"等单字中文）

## API 参考

### `TextPreprocessor(stopwords_path=None)`

主处理器类。

- **process(text)** → `str`：完整流水线处理，返回空格分隔关键词
- **process_with_steps(text)** → `dict`：返回每步中间结果，用于调试
- **is_near_duplicate(text_a, text_b, threshold=0.85)** → `bool`：Jaccard 相似度判重
- **text_fingerprint(text)** → `str`：MD5 指纹

### 工具函数

- `batch_process(texts, preprocessor=None)` → `List[str]`：批量处理
- `deduplicate_corpus(texts, threshold=0.85)` → `List[str]`：语料库去重

## 设计说明

### 为什么广告检测改为 ≥2 次命中？

原始关键词列表包含了"赞助""推广"等词，在新闻类微博中频繁出现（如"美团外卖赞助加纳队"）。改为 2 次命中阈值后：

- 纯新闻提及（仅有"赞助"一个命中）→ 正常保留
- 促销类内容（"限时优惠"+"加V"+"包邮"多个命中）→ 正确过滤

### 为什么 emoji 范围需要精确？

常见的 emoji 正则常复制过宽的范围 `\U000024C2-\U0001F251`，这一范围覆盖了：

- CJK 统一表意文字（U+4E00–U+9FFF，即全部中文）
- 全角符号（U+FF00–U+FFEF）
- 大量非 emoji 的其他 Unicode 区块

直接用这个范围会删除所有中文。本模块已替换为仅覆盖实际 emoji 区块的精确范围。

### 停用词表覆盖范围

`stopwords.txt` 包含 631 个停用词，覆盖：

- 代词（我、你、他、这、那）
- 介词/连词（的、了、在、是、和、因为、所以）
- 语气词（吗、吧、啊、呢）
- 程度副词（很、非常、十分、太、最）
- 时态词（已经、正在、曾经、将要）
- 否定词（不、没、未、别）
- 数量词（一些、许多、每个、所有）
- 方位词（上面、下面、里面、外面）
- 常用虚词组合
