# NLP 事件抽取模块说明

## 模块位置
- 核心代码：`event_extraction.py`
- 依赖：`named_entity_recognition.py`

## 概述
基于**触发词 + 规则 + NER**的传统方法，从文本中识别"发生了什么"，输出结构化事件信息。

| 事件类型 | 触发词 | 要素槽位 |
|----------|--------|----------|
| 产品发布 | 发布、推出、上市、首发 | 时间、主体、产品、地点 |
| 产品召回 | 召回、下架、停售 | 时间、主体、产品、原因 |
| 处罚整治 | 处罚、罚款、整治、查封、拘留 | 时间、主体、原因、金额 |
| 诉讼维权 | 起诉、诉讼、维权、判决 | 时间、原告、被告、事由 |
| 辟谣声明 | 辟谣、否认、声明、回应 | 时间、主体、事由 |
| 事故灾害 | 事故、火灾、地震、伤亡 | 时间、地点、主体、结果 |
| 合作签约 | 合作、签约、战略合作 | 时间、主体、合作方、金额 |
| 获奖荣誉 | 获奖、荣获、夺得 | 时间、主体、奖项 |
| 人事变动 | 任命、离职、辞职、出任 | 时间、主体、职务、机构 |
| 政策发布 | 发布公约、出台政策、颁布 | 时间、主体、政策、内容 |

## 快速使用
```python
from event_extraction import EventExtractor, format_events

ee = EventExtractor()
events = ee.extract("2026年6月，某公司在上海发布新款手机。")
# [{"type": "产品发布", "trigger": "发布",
#   "slots": {"时间": "2026年6月", "地点": "上海",
#             "主体": "某公司", "产品": "新款手机"}}]

# 简化摘要
summary = ee.extract_summary(text)
# [{"事件类型": "产品发布", "时间": "2026年6月", ...}]
```

## 算法原理

### 流程
```
原始文本 → NER(实体识别)
        → 触发词检测(按长度降序)
        → 合并重叠触发词
        → 上下文槽位填充(利用NER实体+启发式规则)
        → 输出结构化事件
```

### 触发词检测
- 预定义10类事件 ~60个触发词
- 按长度降序匹配（长触发词优先）
- 合并重叠触发词

### 槽位填充
- 触发词前后 60 字范围内搜索 NER 实体
- 时间、地点、金额：直接使用对应类型实体
- 主体：机构实体优先，次选人名
- 产品：使用产品名实体
- 原因/事由：从触发词前 30 字提取"因/为/由于"后的内容
- 结果：触发词后 30 字

### 事件类型定义格式
```python
EVENT_DEFINITIONS = {
    "产品发布": {
        "triggers": ["发布", "推出", "上市", ...],
        "slots": ["时间", "主体", "产品", "地点"],
    },
    ...
}
```
