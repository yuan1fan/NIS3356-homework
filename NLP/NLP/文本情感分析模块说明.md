# NLP 情感分析模块说明

## 模块位置
- 核心代码：`sentiment_analysis.py`
- 依赖：`preprocessing.py`, `jieba`

## 概述
基于**情感词典 + 规则**的传统方法，无需标注数据即可分析文本情感倾向。

| 支持能力 | 说明 |
|----------|------|
| 三分类 | 正面 / 中性 / 负面 |
| 连续分数 | -1.0 ~ +1.0 |
| 细粒度 | 喜悦、愤怒、悲伤、焦虑、惊讶、厌恶 |
| 方面级 | 对功能、价格、质量、服务、外观、口感分别判断 |
| 表情符号 | 😊😡😢等情感分析 |

## 算法原理

### 1. 情感词典
| 词典 | 规模 | 作用 |
|------|------|------|
| 正面词 | ~150 词 | 好、棒、喜欢、优秀…… |
| 负面词 | ~150 词 | 差、坏、垃圾、愤怒…… |
| 否定词 | ~20 词 | 不、没、别、无……（翻转情感极性） |
| 程度副词 | ~35 词 | 很(×1.5)、非常(×2.0)、太(×2.0)……（加权） |
| 转折词 | ~10 词 | 但是、然而——加重后半句权重 |
| 情感表情 | ~25 个 | 😊+2.0, 😡-2.5…… |

### 2. 评分规则
```
分句 → 对每个分句逐词扫描
  ├ 正面词: +1.0 × 程度副词权重 × (否定翻转)
  ├ 负面词: -1.0 × 程度副词权重 × (否定翻转)
  ├ 否定词: 翻转后续情感词的极性
  └ 程度词: 放大后续情感词的强度
分句分数 = 求和 / √(分句长度)
转折处理: "但是"后分句权重×2
总分数 = 加权平均，归一化到[-1, 1]
表情符号: 额外贡献×0.3
```

### 3. 方面级分析
```
对每个方面关键词（功能、价格、质量等）：
  ├ 定位关键词在原文中的位置
  └ 提取前后各30字符作为上下文，分析情感
```

## 快速使用
```python
from sentiment_analysis import SentimentAnalyzer
sa = SentimentAnalyzer()

# 整体分析
r = sa.analyze("这个产品功能不错，但是价格太离谱了")
# {"sentiment": "负面", "score": -0.28, "positive_words": ["不错"], "negative_words": ["离谱"]}

# 细粒度情绪
r2 = sa.analyze("太让人愤怒了")
# {"fine_grained": {"愤怒": 1}}

# 方面级分析
ar = sa.analyze_aspects("功能不错，价格太离谱，质量挺好")
# {"aspects": [{"aspect": "功能", "sentiment": "正面", "score": 0.41}, ...],
#  "overall": "偏负面"}
```

## 测试结果
| 输入文本 | 情感 | 分数 |
|----------|------|------|
| 功能不错，价格太离谱 | 负面 | -0.28 |
| 天气真好，心情愉快 | 正面 | +0.50 |
| 愤怒！欺骗消费者 | 负面 | -0.79 |
| 一般般，没感觉 | 中性 | 0.00 |
| 服务质量太差 | 负面 | -0.20 |
| 突发暴雨新闻 | 中性 | 0.00 |
| 太喜欢了，超级好用 | 正面 | +0.67 |

## 监督式方法
`SupervisedSentimentAnalyzer` 封装了朴素贝叶斯分类器，需要已标注的情感数据才能使用。

## 数据流
```
原始文本 → 去噪 → 分句 → 逐词扫描(词典+规则) → 分数汇总 → 情感分类
```
