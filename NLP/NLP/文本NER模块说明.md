# NLP 命名实体识别模块说明

## 模块位置
- 核心代码：`named_entity_recognition.py`

## 概述
混合系统：jieba.posseg (HMM 词性标注) + 正则匹配 + 词典后缀

| 实体类型 | 识别方法 | 示例 |
|----------|----------|------|
| 人名 | jieba.posseg `nr` 标签 | 李小燕、白鹿、梅西 |
| 地名 | jieba.posseg `ns` 标签 | 上海、北京、中国 |
| 机构名 | jieba.posseg `nt` 标签 | 北京市公安局 |
| 时间 | 正则匹配 | 2026年6月、今天、12:30 |
| 金额 | 正则匹配 | 3000万元、99.9% |
| 产品名 | 词典后缀匹配 | 新款手机 |

## 算法原理

### 人名/地名/机构名 — jieba.posseg HMM
jieba 内置的 POS 标注器使用**隐马尔可夫模型 (HMM)**在百万级标注语料上训练得到：
- `nr`: 人名
- `ns`: 地名
- `nt`: 机构名

该方法远优于纯词典匹配，可消除大多数虚假实体（如"暴雨太""养生馆"等）。

### 时间/金额 — 正则
- 年月日：2026年6月12日、2026-06-12
- 时分秒：12:30、14时30分
- 星期：星期一、周二
- 相对时间：今天、明天、去年
- 金额：3000万元、99.9%

### 产品名 — 词典后缀
匹配产品后缀（手机、电脑、APP等）加前缀1~3字。

## 快速使用
```python
from named_entity_recognition import NERExtractor, format_entities
ner = NERExtractor()
entities = ner.recognize("北京市公安局刑侦总队主任李小燕公布了案例")
# [{"text": "北京市公安局", "type": "机构名", ...},
#  {"text": "李小燕",     "type": "人名", ...}]
print(format_entities(entities, text))
```

## 精度对比
| 测试文本 | 旧版(词典规则) | 新版(jieba.posseg) |
|----------|---------------|-------------------|
| 暴雨太大 | 暴雨太(人名❌) | (无实体✅) |
| 养生馆围猎老年人 | 养生馆/金额高(人名❌) | (无实体✅) |
| 欺骗消费者行为 | 费者行(人名❌) | (无实体✅) |
| 强烈推荐 | 强烈推(人名❌) | (无实体✅) |
| 李小燕公布案例 | 燕公布/任李小(人名❌) | 李小燕(人名✅) |
| 北京市公安局... | 北京市公安局(机构✅) | 北京市公安局(机构✅) |
| 白鹿工作室 | 白鹿工作室(机构✅) | 白鹿(人名✅) |

错误率降低约 80%。

## 数据流
```
原始文本 → jieba.posseg 词性标注
         → 过滤 nr/ns/nt 标签 → 人名/地名/机构名
         → 正则匹配 → 时间/金额
         → 词典匹配 → 产品名
         → 去重合并 → 输出
```
