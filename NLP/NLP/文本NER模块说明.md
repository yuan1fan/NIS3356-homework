# NLP 命名实体识别模块说明

## 模块位置
- 核心代码：`named_entity_recognition.py`

## 概述
基于**词典 + 正则规则**的传统方法，无需训练数据即可识别中文文本中的命名实体。

| 实体类型 | 识别方法 | 示例 |
|----------|----------|------|
| 时间 | 正则匹配 | 2026年6月、今天、星期一、12:30 |
| 金额 | 正则匹配 | 3000万元、99.9% |
| 人名 | 姓氏词典 + 2字匹配 + 黑名单 | 李小燕、欧阳修 |
| 地名 | 省市区/国家词典 | 上海、北京、中国 |
| 机构名 | 机构后缀 + 前缀匹配 | 北京市公安局、白鹿工作室 |
| 产品名 | 产品后缀 + 前缀匹配 | 新款手机、iPhone |

## 快速使用
```python
from named_entity_recognition import NERExtractor, format_entities
ner = NERExtractor()
entities = ner.recognize("2026年6月，某公司在上海发布新款手机。")
# entities = [
#   {"text": "2026年6月", "type": "时间", "start": 0, "end": 7},
#   {"text": "某公司",   "type": "机构名", "start": 8, "end": 11},
#   {"text": "上海",     "type": "地名", "start": 13, "end": 15},
#   {"text": "新款手机", "type": "产品名", "start": 17, "end": 21},
# ]
```

## 词典与规则

### 时间
- 年月日: `2026年6月12日`, `2026-06-12`
- 时分秒: `12:30`, `14时30分`
- 星期: 星期一、周二
- 相对时间: 今天、明天、去年、本月

### 金额
- `3000万元`, `99.9%`, `$50`

### 人名
- 常见姓氏约90个（李、王、张、刘等单姓 + 欧阳、司马等复姓）
- 要求姓氏后紧跟2个中文字符
- 黑名单过滤：者、的、了、在、是等常见非名字字符

### 地名
- 34个省级行政区
- 100+个主要城市
- 50+个国家

### 机构名
- 后缀匹配：公司、集团、大学、局、处、部、工作室等30+后缀
- 前缀限制2~6字避免过长匹配

### 产品名
- 后缀匹配：手机、电脑、APP等
- 前缀限制1~3字

## 测试结果
| 输入文本 | 识别出的实体 |
|----------|-------------|
| 2026年6月，某公司在上海发布新款手机 | 时间:2026年6月, 机构名:某公司, 地名:上海, 产品名:新款手机 |
| 今天天气真好，白鹿工作室发表声明 | 时间:今天, 机构名:白鹿工作室 |
| 北京市公安局刑侦总队主任李小燕公布 | 机构名:北京市公安局, 人名:李小燕 |
| 涉案金额高达3000万余元 | 金额:3000万 |
| 央视曝养生馆围猎老年人 | 机构名:央视 |
| 太让人愤怒了 | （无实体） |

## 数据流
```
原始文本 → 依次识别各类实体(时间→金额→人名→地名→机构→产品)
         → 去重合并重叠实体 → 按位置排序 → 输出
```
