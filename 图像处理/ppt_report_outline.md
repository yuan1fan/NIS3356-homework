# 数字图像处理模块汇报文案

---

## 第 1 页：功能和解决的问题

### 模块定位

本模块面向微博热搜、小红书等社交媒体数据中的图片和视频内容，负责从非结构化视觉素材中提取可供后续大模型分析的信息。相比只读取帖子文本，本模块进一步处理图片文字、视频画面文字以及图片本身的语义信息，减少热点事件分析中的信息遗漏。

### 1. 视频内容处理：抽帧与自适应间隔

社交媒体帖子中大量信息并不只存在于配文，很多关键内容会出现在视频字幕、新闻截图、采访画面或现场视频中。为了避免逐帧识别带来的巨大开销，本模块对视频进行抽帧处理：

- 视频时长小于 64 秒时，每 2 秒抽取 1 帧；
- 视频时长较长时，根据视频总时长自适应调整间隔，最多抽取 32 帧；
- 对抽取帧进行 OCR 和视觉语义分析；
- 对相邻帧重复文字进行去重，避免同一字幕被重复输入给大模型。

这样既能覆盖视频中的主要文字信息，又能控制计算量，适合批量处理热搜帖子中的短视频素材。

### 2. OCR 识别去噪与分句处理

社交媒体图片中常见水印、账号名、来源标识、压缩噪声、换行断句等问题。直接将 OCR 原始结果交给大模型，会带来冗余和误导。本模块在 PaddleOCR 识别后增加文本清洗与结构优化：

- 去除常见水印和来源标识，例如短小的 `@账号名`、重复媒体署名等；
- 对 OCR 结果中的无意义碎片进行过滤；
- 合并相邻文本块，减少 OCR 框切割导致的碎片化；
- 对因图片换行造成的错误断句进行修复，使结果更接近自然段文本。

最终输出更适合大模型消费的 `ocr_text_compact` 字段，而不是包含大量坐标框的原始 OCR 结果。

### 3. 保留图像语义信息：分类 + 一句话描述

仅有文字信息不足以表达图片的全部内容。例如现场照片、人物照片、事故画面、风景插图、表情包等图片，可能文字很少甚至没有文字，但仍然具有重要语义。为此模块加入视觉语义分析：

- 使用图像分类判断图片类型，例如聊天记录截图、微博截图、新闻现场照片、普通人物照片、风景环境照片、信息图表、宣传海报、表情包等；
- 生成视觉语义标签，例如诈骗、违法犯罪、公共安全、娱乐明星、社会民生、争议舆情等；
- 对适合视觉描述的图片自动生成一句话描述；
- 对聊天记录、社交媒体截图、新闻截图等文字密集图片，优先保留 OCR 文字，不强制生成视觉描述。

这样可以在后续汇总分析中同时保留“图片上写了什么”和“图片大概表达了什么”。

## 第 2 页：简易模块流程图

### 总体流程

```text
输入图片 / 视频
      |
      v
媒体类型判断
      |
      +---------------------------+
      |                           |
      v                           v
图片预处理                    视频抽帧
原图 / 增强 / 去噪             自适应间隔，最多 32 帧
      |                           |
      +-------------+-------------+
                    |
                    v
              PaddleOCR 文字识别
                    |
                    v
        多版本结果评分，选择最佳 OCR 结果
                    |
                    v
      OCR 文本去噪、水印过滤、重复文本压缩、分句修复
                    |
                    v
           Chinese-CLIP 图像分类与语义标签
                    |
                    v
       根据图片类型决定是否生成一句话视觉描述
                    |
                    v
       汇总输出 JSON：OCR 文本 + 置信度 + 图像语义 + 描述
```

### 对应简化链路

```text
输入 -> OCR -> 分类、去噪 -> 一句话描述 -> 汇总输出
```

### 输出内容

模块最终输出面向后续大模型汇总分析的结构化 JSON，主要字段包括：

- `ocr_text_compact`：压缩清洗后的 OCR 文本；
- `quality_assessment`：OCR 平均置信度、文本块数量、是否需要复核等；
- `best_variant`：当前图片选择的最佳预处理版本；
- `visual_semantics.visual_type`：图片或视频帧的视觉类型；
- `visual_semantics.semantic_tags`：视觉语义标签；
- `visual_semantics.image_caption`：必要时生成的一句话视觉描述；
- `frame_sampling`：视频抽帧间隔、抽帧数量、时间戳等信息。

## 第 3 页：具体技术路线与使用模型

### 1. OCR 文字识别

使用模型与工具：

- PaddleOCR：用于中文图片文字检测与识别；
- OpenCV：用于图像读取、灰度化、增强、去噪、锐化、裁剪与视频抽帧；
- 多版本图像预处理：原图、灰度图、CLAHE 对比度增强、锐化、去噪、二值化等。

技术思路：

- 同一张图片生成多个预处理版本；
- 每个版本分别送入 PaddleOCR；
- 根据平均置信度、识别文本长度、文本块数量、低置信度比例等指标综合评分；
- 自动选择最佳 OCR 版本作为最终结果；
- 输出识别文本、置信度、最佳预处理版本和质量评估结果。

### 2. 视频文本提取

使用工具：

- OpenCV 视频读取；
- PaddleOCR 帧文字识别；
- 文本去重与合并规则。

技术思路：

- 先读取视频元信息，获取视频时长；
- 若视频较短，则按 2 秒间隔抽帧；
- 若视频较长，则按 `视频时长 / 32` 自适应设置抽帧间隔；
- 每个视频最多抽取 32 帧，避免长视频处理时间过长；
- 对抽帧结果执行 OCR；
- 对相邻帧重复字幕或重复画面文字进行压缩，只保留有效变化内容。

### 3. OCR 后处理与文本清洗

使用方法：

- 水印和账号来源过滤规则；
- OCR 文本块合并；
- 重复文本压缩；
- 轻量级断句修复逻辑。

技术思路：

- 删除明显不是正文的短水印，例如 `@北京日报`、重复来源署名等；
- 过滤过短、重复、噪声化的文本片段；
- 对因图片换行导致的“老；年人”“犯；疑人”等异常断句进行修复；
- 生成适合大模型直接读取的紧凑文本字段。

### 4. 图像语义分类

使用模型：

- Chinese-CLIP：`OFA-Sys/chinese-clip-vit-base-patch16`

技术思路：

- 将图片与多组中文候选标签进行图文匹配；
- 通过 zero-shot 方式判断图片类型，不需要额外训练分类模型；
- 当前支持的视觉类型包括聊天记录截图、新闻报道截图、微博或社交媒体截图、新闻现场照片、普通人物照片、风景或环境照片、信息图表、宣传海报、商品广告图片、表情包或梗图、证件票据截图、视频画面截图等；
- 同时输出社会安全相关语义标签，例如诈骗、违法犯罪、公共安全、医疗健康、社会民生、争议舆情等。

### 5. 一句话视觉描述

使用模型：

- BLIP Image Captioning：`Salesforce/blip-image-captioning-base`

技术思路：

- 不是所有图片都需要生成 caption；
- 对聊天记录、微博截图、新闻截图等文字密集图片，OCR 文本已经是主要信息来源；
- 对新闻现场照片、人物照片、风景环境照片、视频画面截图、表情包等视觉语义更强的图片，生成一句话描述；
- 将描述结果合并到 `visual_semantics.image_caption` 中，补充 OCR 无法覆盖的视觉信息。

### 6. 最终输出给大模型的信息

模块最终不直接把全部 OCR 坐标框交给大模型，而是输出压缩后的结构化结果：

```json
{
  "media_type": "image_or_video",
  "ocr_text_compact": "清洗、去重、分句修复后的文字内容",
  "quality_assessment": {
    "avg_confidence": 0.95,
    "needs_review": false
  },
  "visual_semantics": {
    "visual_type": "新闻现场照片",
    "semantic_tags": ["社会民生", "公共安全"],
    "image_caption": "一句话视觉描述"
  }
}
```

该设计的核心目标是：把图片和视频转化为大模型容易理解的结构化信息，同时尽量保留文字、置信度和视觉语义三类关键信息。
