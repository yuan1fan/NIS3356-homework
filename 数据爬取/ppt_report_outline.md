# 微博热搜数据爬取模块汇报文案

---

## 第 1 页：功能和解决的问题

### 模块定位

本模块面向微博热搜场景，负责从实时热搜榜中抓取可用于后续分析的代表性帖子数据。它不是只保存热搜排行榜，而是进入每个热搜词条，提取对应代表帖的正文、图片、视频和音频信息，为 ASR、NLP、图像处理、数据分析和大模型汇总提供统一输入。

### 1. 固定实时热搜快照，避免重复和漏抓

微博热搜 Top50 是实时刷新的。如果每处理一个词条就重新访问热搜榜，排名和词条可能已经变化，容易导致重复抓取或遗漏词条。

本模块的处理方式是：

- 先一次性抓取当前时刻微博热搜 Top50；
- 将热搜词、排名、热度、标签和详情页链接保存到 `hot_topics_snapshot.jsonl`；
- 后续爬取过程始终按照这份快照逐条处理；
- 保证一次任务中的 50 个热搜词条是同一时间基准下的数据。

### 2. 进入热搜详情页，抓取代表帖子

课程任务关注的是热点新闻或热点事件本身，而不是热搜榜单截图。因此模块会进入每个热搜词条的搜索结果页，抓取候选微博帖子，并从中选择一个代表帖。

代表帖中提取的信息包括：

- 帖子正文文本；
- 发帖用户、发布时间、转发数、评论数、点赞数；
- 图片 URL 和本地图片文件；
- 视频 URL 和本地视频文件；
- 音频 URL 和本地音频文件；
- 原始帖子链接，便于人工复核。

### 3. 质量筛选，减少低价值数据

热搜详情页里并不是每条微博都适合作为代表帖。例如有些帖子没有图片或视频，有些只有几个字，有些视频过长，不适合后续批量处理。

本模块设置了代表帖筛选规则：

- 默认正文长度至少 12 个字符；
- 默认要求帖子包含图片、视频或音频；
- 默认跳过超过 20 分钟的视频；
- 在满足条件的候选中，优先选择媒体更完整、文本更充足、互动量更高的帖子；
- 对被过滤的候选帖记录拒绝原因，例如 `text_too_short`、`no_media`、`video_too_long`。

### 4. 为后续多模态模块提供原始数据

最终输出不是单一文本，而是结构化帖子数据和本地媒体文件：

- NLP 模块可以读取代表帖正文；
- 图像处理模块可以读取图片并进行 OCR、分类和视觉描述；
- ASR 模块可以读取视频或音频并转写语音内容；
- 数据分析模块可以统计热搜排名、热度、媒体数量和候选过滤情况；
- 大模型可以汇总文本、图片 OCR、视频字幕和语义标签，形成热点事件分析结果。

## 第 2 页：简易模块流程图

### 总体流程

```text
输入微博登录 Cookie / 热搜榜入口
              |
              v
请求微博热搜接口
固定当前 Top50 快照
              |
              v
保存 hot_topics_snapshot.jsonl
              |
              v
按快照顺序逐个处理热搜词条
              |
              v
构造热搜详情页搜索 URL
优先抓取 s.weibo.com 搜索结果
              |
              v
解析候选微博帖子
正文 / 用户 / 时间 / 指标 / 图片 / 视频 / 音频
              |
              v
候选帖去重与质量筛选
文本过短、无媒体、视频过长则跳过
              |
              v
选择代表帖
媒体完整度 + 文本长度 + 互动量综合排序
              |
              v
下载图片、视频、音频到本地 media 目录
              |
              v
输出 representative_posts.jsonl + summary.json
```

### 对应简化链路

```text
热搜快照 -> 词条详情页 -> 候选帖子解析 -> 代表帖筛选 -> 媒体下载 -> 结构化输出
```

### 数据流向

```text
微博热搜数据
    |
    +-- 帖子正文 ----------> NLP / 大模型
    |
    +-- 图片文件 ----------> 图像处理 OCR / 图像分类 / 视觉描述
    |
    +-- 视频文件 ----------> 视频抽帧 OCR / ASR
    |
    +-- 排名热度指标 ------> 数据分析 / 趋势分析
```

## 第 3 页：具体技术路线

### 1. 登录态与请求管理

使用工具：

- `requests.Session`：保持统一请求会话；
- 微博登录 Cookie：访问需要登录态的热搜详情和搜索结果；
- 自定义请求头：模拟正常浏览器访问；
- 请求间隔和重试机制：降低请求失败率，避免访问过快。

实现方式：

- Cookie 通过命令行参数、环境变量或本地文件传入；
- 不把 Cookie 写入代码，也不提交到 Git；
- 每次运行创建独立时间戳输出目录，便于保留历史数据。

### 2. 热搜榜快照抓取

使用接口：

```text
https://weibo.com/ajax/side/hotSearch
```

处理内容：

- 提取热搜词条排名 `rank`；
- 提取热搜词 `word` 和搜索词 `word_scheme`；
- 提取热度值 `raw_hot_score`；
- 提取热搜标签 `label`；
- 生成对应详情页链接 `detail_url`；
- 保存为 `hot_topics_snapshot.jsonl`。

这一设计保证后续处理不会受到热搜实时刷新影响。

### 3. 热搜详情页帖子解析

优先访问 PC 端微博搜索页：

```text
https://s.weibo.com/weibo?q=关键词&t=31&Refer=top&band_rank=排名
```

解析内容：

- 从搜索结果 HTML 中切分微博卡片；
- 清洗 HTML 标签，得到纯文本正文；
- 提取微博 ID、用户 ID、用户名和发布时间；
- 提取图片链接，并优先转成大图地址；
- 提取视频链接和视频时长；
- 提取转发数、评论数、点赞数；
- 对重复帖子按微博 ID 或原始链接去重。

如果 PC 端搜索结果解析不到候选帖，则使用移动端搜索接口作为兜底：

```text
https://m.weibo.cn/api/container/getIndex
```

### 4. 代表帖选择策略

每个热搜词条可能对应多条候选微博。本模块通过 `SelectionPolicy` 进行筛选和排序。

默认筛选规则：

- `min_text_len = 12`：正文太短则跳过；
- `require_media = true`：没有图片、视频或音频则跳过；
- `max_video_duration_seconds = 1200`：超过 20 分钟的视频跳过；
- `prefer_media = true`：优先选择媒体数量更多的帖子。

排序依据：

```text
媒体数量 -> 文本长度 -> 互动量 -> 搜索结果中的靠前位置
```

其中互动量计算方式为：

```text
engagement_score = 转发数 * 3 + 评论数 * 2 + 点赞数
```

### 5. 媒体下载与文件组织

媒体下载内容：

- 图片：保存到 `media/<排名_热搜词>/images/`；
- 视频：保存到 `media/<排名_热搜词>/videos/`；
- 音频：保存到 `media/<排名_热搜词>/audios/`。

文件命名方式：

- 根据媒体 URL 计算哈希；
- 自动保留常见扩展名，例如 `.jpg`、`.mp4`；
- 对无法判断扩展名的媒体保存为 `.bin`；
- 单个媒体默认限制最大 80MB，避免误下载过大文件。

## 第 4 页：输出结果与下游对接

### 输出目录结构

每次运行会生成一个独立时间戳目录，例如：

```text
outputs/20260612_141245/
  hot_topics_snapshot.jsonl
  representative_posts.jsonl
  summary.json
  media/
    01_热搜词/
      images/
      videos/
      audios/
```

### 核心输出文件

`hot_topics_snapshot.jsonl`：

- 保存一次任务开始时的热搜 Top50 快照；
- 每行对应一个热搜词条；
- 主要字段包括排名、词条、热度、标签和详情页链接。

`representative_posts.jsonl`：

- 保存每个热搜词条最终选出的代表帖；
- 每行对应一个热搜词条；
- 包含代表帖正文、用户信息、互动指标、图片、视频、音频、本地媒体路径；
- 同时记录未选中候选帖及其过滤原因。

`summary.json`：

- 记录本次任务的热搜数量、成功选帖数量和筛选策略；
- 便于快速检查一次爬取是否完整。

### 一次真实运行结果

在 `outputs/20260612_141245/` 这次运行中：

- 固定热搜词条数：50；
- 成功选择代表帖：50；
- 筛选策略：文本不少于 12 字，必须包含媒体，跳过超过 20 分钟的视频；
- 下载图片：121 张 `.jpg`；
- 下载视频：30 个 `.mp4`；
- 另有 28 个 `.bin`，主要是无法从 URL 判断真实扩展名的媒体或视频页面资源。

### 模块价值总结

本模块解决的是“从实时热搜榜到可分析多模态数据”的问题。它将微博热搜中的非结构化网页内容转化为稳定、可复现、可交给后续算法模块处理的结构化数据，为热点主题分类、趋势预测和内容安全分析提供基础数据来源。
